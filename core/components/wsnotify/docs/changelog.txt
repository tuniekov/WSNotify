WSNotify Changelog
==================

Version 1.0.0-beta (2025-08-10)
--------------------------------
- Первый релиз компонента WSNotify
- Базовая функциональность WebSocket уведомлений
- Поддержка множественных MODX сайтов
- Управление каналами через gtsapipackages
- API endpoints для аутентификации и синхронизации каналов
- WebSocket клиент на vanilla JavaScript
- Сниппет для инициализации клиента
- Методы отправки уведомлений:
  * sendToUsers() - конкретным пользователям
  * sendToGroups() - группам пользователей
  * sendToChannels() - в каналы
  * sendToAnonymous() - анонимным пользователям
- Автоматическое переподключение при разрыве соединения
- Поддержка браузерных уведомлений
- Интеграция с toast библиотеками
- CSS стили для уведомлений
- Документация и примеры использования
- Настройки компонента через системные настройки MODX

Особенности архитектуры:
- Один WebSocket сервер для множественных MODX сайтов
- Пространства имен Socket.IO для изоляции сайтов
- Конфигурация сайтов в config.json на WebSocket сервере
- Каждый MODX сайт управляет только своими каналами
- API ключи для безопасности
- Логирование операций

Требования:
- MODX Revolution 2.6+
- Node.js для WebSocket сервера
- Socket.IO библиотека
- gtsapipackages для управления каналами
